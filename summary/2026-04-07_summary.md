## 项目概述

基于 LangChain + Chroma 向量数据库 + DashScope（阿里云）API 的 RAG（检索增强生成）应用，提供两个 Streamlit 界面：问答系统和知识库文件上传。

## 开发内容总览

本次开发将项目从单一阿里云通义千问模型的硬编码依赖，改造为统一使用 OpenAI 兼容 API 的通用架构。用户只需修改 `config.py` 中的 `base_url` 和 API Key 即可切换任意模型 Provider。

## 详细代码变更

### 🎯 核心功能模块

#### 模型工厂
- **文件**: `model_factory.py` (新增)
- **变更类型**: 新增
- **变更内容**:
  - 提供 `create_chat_model()` 和 `create_embedding_model()` 函数

- **文件**: `config.py`
- **变更类型**: 修改

  - Embedding Model: `EMBEDDING_MODEL_NAME`, `EMBEDDING_MODEL_BASE_URL`, `EMBEDDING_MODEL_API_KEY`
  - 默认使用通义千问 API 端点
- **影响范围**: 所有使用模型的模块

#### RAG 服务
- **文件**: `rag.py`
- **变更类型**: 修改
- **变更内容**:
  - 移除硬编码的 `ChatTongyi` 和 `DashScopeEmbeddings`
  - 改用 `model_factory.create_chat_model()` 和 `model_factory.create_embedding_model()`
- **影响范围**: 问答系统核心逻辑

## 技术亮点

- **统一 OpenAI 兼容接口**: 所有模型通过 `base_url` 配置，无需区分 Provider 类型
- **零依赖特定 SDK**: 移除了 `langchain_community.chat_models.tongyi`，仅依赖 `langchain-openai`
- **配置极简**: 切换模型只需修改 3-4 个配置项

## 使用示例

### 通义千问（默认）
```python
CHAT_MODEL_NAME = "qwen3-max"
CHAT_MODEL_BASE_URL = "https://dashscope.aliyuncs.com/compatible-mode/v1"
CHAT_MODEL_API_KEY = None  # 从环境变量 DASHSCOPE_API_KEY 读取
```

### vLLM 公网部署
```python
CHAT_MODEL_NAME = "your-model-name"
CHAT_MODEL_BASE_URL = "http://your-vllm-ip:8000/v1"
CHAT_MODEL_API_KEY = "your-api-key"
```

### 切换 Embedding Provider
更换后需清空 `chroma_db/` 重新索引（向量维度可能不同）。
